# Output Documents

This folder is the **output destination** for documents generated by AI agent skills and workflows.

---

## Structure

```
.agents/documents/
│
├── _templates/              # Template files (copy & fill)
├── _examples/               # Filled example documents (reference)
│
├── requirements/            # PRE-IMPLEMENTATION — what to build
│   ├── brd/                 # Business Requirements Document
│   ├── prd/                 # Product Requirements Document
│   ├── fsd/                 # Functional Specification Document
│   ├── tdd/                 # Technical Design Document
│   ├── user-stories/        # User stories per feature
│   └── acceptance-criteria/ # Acceptance criteria per feature
│
├── design/                  # HOW to build — pre-coding design
│   ├── architecture/        # System architecture, C4, component diagrams
│   ├── domain/              # Domain models, bounded contexts
│   ├── database/            # Schema plans, ERDs
│   └── api/                 # API contract design
│
├── application/             # POST-IMPLEMENTATION — documentation of existing code
│   ├── modules/             # Module documentation
│   ├── api/                 # API specifications
│   └── testing/             # Test scenarios
│
├── tasks/                   # EXECUTION — what to do and when
│   ├── backlog/             # Implementation task lists
│   ├── sprints/             # Sprint plans
│   └── roadmap/             # Product roadmap
│
├── operations/              # OPS — running in production
│   ├── runbooks/            # Runbooks
│   └── changelogs/          # Changelogs & release notes
│
└── decisions/               # ADRs — Architecture Decision Records
```

---

## Path Conventions

| Output Type | Path | Generated By |
| :---------- | :--- | :----------- |
| BRD | `requirements/brd/{project-slug}.md` | `idea-validation/stakeholder-requirement-extraction` |
| PRD | `requirements/prd/{project-slug}.md` | `idea-validation/mvp-scope-definition` |
| FSD | `requirements/fsd/{feature-slug}.md` | `planning/user-story-generation`, `planning/acceptance-criteria-writing` |
| TDD | `requirements/tdd/{feature-slug}.md` | `planning/architecture-planning`, `planning/database-schema-planning` |
| User stories | `requirements/user-stories/{feature-slug}.md` | `planning/user-story-generation` |
| Acceptance criteria | `requirements/acceptance-criteria/{feature-slug}.md` | `planning/acceptance-criteria-writing` |
| Architecture plan/doc | `design/architecture/` | `planning/architecture-planning`, `documentation/architecture-documentation` |
| Domain model | `design/domain/` | `planning/domain-modeling` |
| Database schema | `design/database/` | `planning/database-schema-planning` |
| API contract | `design/api/{module-slug}/` | `planning/api-contract-design` |
| Module documentation | `application/modules/{module-slug}/` | `documentation/module-documentation` |
| API specification | `application/api/{module-slug}/` | `documentation/api-documentation` |
| Test scenarios | `application/testing/{module-slug}/` | `testing/qa-design`, `testing/test-case-design` |
| Implementation tasks | `tasks/backlog/{feature-slug}.md` | `planning/feature-decomposition` |
| Sprint plan | `tasks/sprints/sprint-{N}.md` | `planning/sprint-planning` |
| Roadmap | `tasks/roadmap/roadmap.md` | `planning/roadmap-creation` |
| Runbook | `operations/runbooks/` | `documentation/runbook-creation` |
| Changelog | `operations/changelogs/` | `documentation/changelog-generation` |
| Release notes | `operations/changelogs/` | `documentation/release-notes-generation` |
| ADR | `decisions/` | `documentation/adr-writing` |

> All paths are relative to `.agents/documents/`
